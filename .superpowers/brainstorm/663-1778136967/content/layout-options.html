用户手册系统 — 前端布局方案

用户扫码后看到的页面，三种布局对比

A. 经典文档布局

截图占位

左侧边栏目录 + 右侧内容区，类似 VuePress / GitBook 风格

优点

  • 目录导航清晰
  • 适合长篇文档
  • 用户直觉成本低

缺点

  • 手机屏窄时侧栏占空间

B. 卡片式首页 + 详情页

首页用卡片展示手册分类，点击进入详情阅读页（详情页可用 A 或 C 布局）

优点

  • 视觉直观、友好
  • 适合多手册场景
  • 扫码直达内容 + 可探索

缺点

  • 多一层页面跳转

C. 纯阅读布局（移动优先）

📱 截图

聚焦内容阅读，顶部/底部导航，类似微信公众号文章风格

优点

  • 移动端体验最佳
  • 加载最快
  • 扫码即读无干扰

缺点

  • 目录浏览不方便
  • 不适合多章节导航
